# 脚本系统实现计划（完整版）

> 基于 `Rust动态加载CSharp示例.md` 设计，实现 Rust ↔ C# 双向调用脚本系统

---

## 确认的设计决策

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 实现范围 | Phase 4（高级特性） | 基础框架 + 交互 + 自动化 + 高级特性 |
| 项目位置 | `engine/scripting/` | 作为引擎子系统 |
| 反射方案 | 混合方案 | Mono 反射 + C# 自注册接口 |
| 回调机制 | Phase 1 基础功能 | 不作为独立 Phase |
| 闭包支持 | 必须支持 | Lambda 捕获外部变量 |
| 异步方案 | 方案 C：混合方案 | 同步 + 异步 + 长时任务三种 |
| 异步场景 | 全场景 | 资源加载 + 网络请求 + 长时计算 |
| 错误传播 | Result<T> 类型 | ScriptValue 包含 error_flag |
| ScriptValue | 固定结构体 | 避免堆分配，零 GC 压力 |
| 回调描述 | 必须提供 | 用于调试/日志/错误追踪 |
| 测试优先级 | 闭包验证 > 双向调用 > 错误处理 | - |

## 1. 核心类型系统

### 1.1 ScriptValue 固定结构体

```rust
// value_bridge.rs
#[repr(C, packed)]
pub struct ScriptValue {
    pub type_tag: u8,          // 类型标识
    pub error_flag: u8,        // Result<T> 标志 (0=Ok, 1=Err)
    pub reserved: u16,         // 对齐保留
    pub int_value: i32,
    pub float_value: f32,
    pub double_value: f64,
    pub ptr_value: *mut c_void,
}

pub enum ScriptTypeTag {
    Null      = 0,
    Int       = 1,
    Float     = 2,
    Double    = 3,
    String    = 4,
    Object    = 5,
    Array     = 6,
    AsyncOp   = 7,
    TaskId    = 8,
}

impl ScriptValue {
    pub fn ok(value: ScriptValue) -> Self;
    pub fn err(message: String) -> Self;
    pub fn is_ok(&self) -> bool;
    pub fn is_err(&self) -> bool;
    pub fn get_error_message(&self) -> Option<String>;
}
```

### 1.2 回调类型系统（三种）

```rust
// callback_types.rs

// 类型 1: 同步回调（立即返回）
pub type SyncCallback = extern "C" fn(ScriptValue) -> ScriptValue;

// 类型 2: 异步回调（完成通知）
pub type AsyncCallback = extern "C" fn(ScriptValue, *mut c_void);

// 类型 3: 长时任务（进度+完成）
pub type TaskCallback = extern "C" fn(
    ScriptValue,
    *mut c_void,  // progress_ptr
    *mut c_void,  // completion_ptr
);

pub struct CallbackDescriptor {
    pub name: String,
    pub callback_type: CallbackType,
    pub description: String,
    pub signature: String,
    pub is_closure: bool,
    pub priority: u32,
}

pub enum CallbackType {
    Sync,
    Async,
    Task,
}
```

### 1.3 回调注册表结构

```rust
// callback_registry.rs
pub struct CallbackRegistry {
    sync_callbacks: HashMap<String, SyncCallbackEntry>,
    async_callbacks: HashMap<String, AsyncCallbackEntry>,
    task_callbacks: HashMap<String, TaskCallbackEntry>,
    running_tasks: HashMap<TaskId, RunningTask>,
    async_runtime: AsyncRuntime,
}

pub struct SyncCallbackEntry {
    callback: SyncCallback,
    descriptor: CallbackDescriptor,
    closure_context: Option<*mut c_void>,
}

pub struct AsyncCallbackEntry {
    callback: AsyncCallback,
    descriptor: CallbackDescriptor,
    closure_context: Option<*mut c_void>,
}

pub struct TaskCallbackEntry {
    callback: TaskCallback,
    descriptor: CallbackDescriptor,
    closure_context: Option<*mut c_void>,
    supports_progress: bool,
}

pub struct RunningTask {
    id: TaskId,
    progress: f32,
    status: TaskStatus,
    result: Option<ScriptValue>,
}

pub enum TaskStatus {
    Running,
    Completed,
    Failed,
    Cancelled,
}
```

## 2. API 设计

### 2.1 Rust 公共 API

```rust
impl ScriptManager {
    // 生命周期
    pub fn new() -> Result<Self, ScriptError>;
    pub fn shutdown(&mut self);
    
    // DLL 管理
    pub fn load_script(&mut self, path: &str) -> Result<String, ScriptError>;
    pub fn unload(&mut self, name: &str);
    pub fn reload(&mut self, path: &str) -> Result<String, ScriptError>;
    
    // Rust → C# 调用
    pub fn execute(&self, assembly, ns, class, method, args) -> Result<ScriptValue>;
    
    // 同步回调
    pub fn register_sync_callback(&mut self, name, callback, descriptor);
    pub fn trigger_sync(&self, name, arg) -> Result<ScriptValue>;
    
    // 异步回调
    pub fn register_async_callback(&mut self, name, callback, descriptor);
    pub fn trigger_async(&self, name, arg) -> impl Future<Output = Result<ScriptValue>>;
    
    // 长时任务
    pub fn register_task_callback(&mut self, name, callback, descriptor);
    pub fn trigger_task(&self, name, arg) -> Result<TaskId>;
    pub fn query_task_progress(&self, id) -> Option<f32>;
    pub fn query_task_result(&self, id) -> Option<Result<ScriptValue>>;
    pub fn cancel_task(&self, id) -> bool;
    
    // 回调管理
    pub fn unregister_callback(&mut self, name);
    pub fn list_callbacks(&self) -> Vec<CallbackDescriptor>;
}
```

## 3. C# 端设计

### 3.1 PluginBase.cs

```csharp
public abstract class ScriptPlugin
{
    // 同步回调注册（支持闭包）
    protected void RegisterSyncCallback<TArg, TRet>(
        string name,
        Func<TArg, TRet> handler,
        string description = "",
        string signature = ""
    );
    
    // 异步回调注册
    protected void RegisterAsyncCallback<TArg, TRet>(
        string name,
        Func<TArg, Task<TRet>> handler,
        string description = ""
    );
    
    // 长时任务注册
    protected void RegisterTaskCallback<TArg, TRet>(
        string name,
        Func<TArg, TaskContext, Task<TRet>> handler,
        string description = "",
        bool supportsProgress = true
    );
    
    // Thunk 实现（闭包支持）
    [UnmanagedCallersOnly]
    private static ScriptValue SyncThunk<TArg, TRet>(ScriptValue arg, IntPtr context);
    [UnmanagedCallersOnly]
    private static void AsyncThunk<TArg, TRet>(ScriptValue arg, IntPtr completion, IntPtr context);
    [UnmanagedCallersOnly]
    private static void TaskThunk<TArg, TRet>(ScriptValue arg, IntPtr progress, IntPtr completion, IntPtr context);
}

public class TaskContext
{
    public Action<float> ReportProgress { get; internal set; }
}
```

### 3.2 ScriptValue.cs

```csharp
[StructLayout(LayoutKind.Sequential, Pack = 1)]
public struct ScriptValue
{
    public byte TypeTag;
    public byte ErrorFlag;      // 0=Ok, 1=Err
    public ushort Reserved;
    public int IntValue;
    public float FloatValue;
    public double DoubleValue;
    public IntPtr PtrValue;
    
    // Ok 构造
    public static ScriptValue Ok<T>(T value);
    
    // Err 构造
    public static ScriptValue Err(string message);
    
    // Result<T> 检查
    public bool IsOk => ErrorFlag == 0;
    public bool IsErr => ErrorFlag == 1;
    public string GetErrorMessage();
    
    // 类型转换
    public T GetValue<T>();
    public static ScriptValue From<T>(T value);
}
```

## 4. 使用场景示例

### 4.1 同步回调（闭包捕获）

```csharp
public class GameLogicPlugin : ScriptPlugin
{
    private int playerScore = 100;
    
    public override void OnLoad() {
        RegisterSyncCallback<int, int>(
            "CalculateScore",
            (baseScore) => baseScore * 2 + playerScore,  // 捕获 playerScore
            description: "Player score calculation"
        );
    }
}
```

```rust
let result = manager.trigger_sync("CalculateScore", ScriptValue::Int(10));
if result.is_ok() {
    println!("Score: {}", result.int_value);
}
```

### 4.2 异步回调（资源加载）

```csharp
public class ResourcePlugin : ScriptPlugin
{
    public override void OnLoad() {
        RegisterAsyncCallback<string, Texture>(
            "LoadTexture",
            async (path) => await AssetLoader.LoadAsync(path),
            description: "Async texture loader"
        );
    }
}
```

```rust
let future = manager.trigger_async("LoadTexture", ScriptValue::String("hero.png"));
let result = future.await?;
```

### 4.3 长时任务（网络请求 + 进度）

```csharp
public class NetworkPlugin : ScriptPlugin
{
    public override void OnLoad() {
        RegisterTaskCallback<string, byte[]>(
            "DownloadFile",
            async (url, context) => {
                for (int i = 0; i < 100; i++) {
                    // 模拟下载
                    context.ReportProgress(i / 100.0f);
                    await Task.Delay(100);
                }
                return new byte[1024];
            },
            description: "File downloader with progress",
            supportsProgress: true
        );
    }
}
```

```rust
let task_id = manager.trigger_task("DownloadFile", ScriptValue::String("http://..."))?;
loop {
    let progress = manager.query_task_progress(task_id);
    if progress >= 1.0 { break; }
}
let result = manager.query_task_result(task_id)?;
```

## 7. 技术风险与缓解

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| 闭包数据 GC 回收 | 回调失效 | GCHandle 固定委托对象 |
| 异步完成通知遗漏 | Future 永不完成 | 超时机制 + 错误日志 |
| Task→Future 桥接复杂 | 实现困难 | completion_ptr 简化桥接 |
| 进度通知频繁 | 性能影响 | 进度阈值过滤（<0.01 忽略） |
| Mono 反射不完整 | 无法获取签名 | C# PluginInfo 自注册优先 |
| Assembly 无法卸载 | 内存泄漏 | 定期清理提示重启 |

## 12. 实现思路与技术细节（Phase 1 实际实现）

### 12.1 核心架构决策

#### 为什么不用 Mono 嵌入？

**问题**：`wrapped_mono` 要求 Mono 运行时只能初始化一次，且需要 `mscorlib.dll` 等依赖文件在特定路径。

**解决方案**：采用 **轻量级 FFI 模式**
- Rust 不嵌入 Mono，只提供回调注册和触发机制
- C# 通过 `DllImport` 调用 Rust DLL
- 双向通信通过函数指针实现

**优势**：
1. 避免单进程 Mono 初始化限制
2. 无需配置 Mono 环境路径
3. 简化编译和部署
4. 单元测试可独立运行（不依赖 Mono）

---

### 12.2 闭包捕获实现方案

#### C# 闭包 → Rust 回调的技术难点

**问题**：`[UnmanagedCallersOnly]` 方法不能是泛型，也不能直接捕获变量。

**解决方案**：**GCHandle + 静态 Thunk**

```
┌─────────────────────────────────────────────────────────────┐
│  C# 注册闭包                                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Func<int, int> handler = x => x * _multiplier;     │   │
│  │                                                      │   │
│  │  // 1. GCHandle 包装 delegate                        │   │
│  │  GCHandle handle = GCHandle.Alloc(handler);          │   │
│  │  nuint context = GCHandle.ToIntPtr(handle);          │   │
│  │                                                      │   │
│  │  // 2. 注册到 Rust                                    │   │
│  │  scripting_register_sync_callback(                   │   │
│  │      manager, "multiply", &SyncThunk, desc, sig,     │   │
│  │      context  // ← 传递 GCHandle 指针                │   │
│  │  );                                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  Rust 触发回调                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  // 存储时：entry.closure_context = context          │   │
│  │                                                      │   │
│  │  // 触发时：                                          │   │
│  │  (entry.callback)(arg, entry.closure_context)        │   │
│  │  //              ↑ arg   ↑ 传回 context              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  C# Thunk 执行                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  [UnmanagedCallersOnly]                              │   │
│  │  static ScriptValue SyncThunk(arg, context) {        │   │
│  │      // 3. 从 context 恢复 delegate                   │   │
│  │      GCHandle handle = GCHandle.FromIntPtr(context); │   │
│  │      Func<...> handler = (Func<...>)handle.Target;   │   │
│  │                                                      │   │
│  │      // 4. 执行闭包（捕获变量可用）                     │   │
│  │      return handler(arg);                            │   │
│  │  }                                                   │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**关键点**：
1. `GCHandle.Alloc(handler)` 防止 GC 回收 delegate
2. `context` 参数传递 GCHandle 指针（作为 `usize` / `nuint`）
3. 静态 Thunk 从 context 恢复 delegate，执行闭包
4. 闭包捕获的外部变量在 delegate 对象中，随 GCHandle 一起传递

### 12.3 三种回调类型签名

```rust
// callback_types.rs

// 同步：立即返回结果
pub type SyncCallback = extern "C" fn(ScriptValue, usize) -> ScriptValue;
//                                    ↑ arg      ↑ context

// 异步：完成后通知
pub type AsyncCallback = extern "C" fn(ScriptValue, usize, usize);
//                                          ↑ arg   ↑ ctx  ↑ manager_ptr

// 任务：带进度通知
pub type TaskCallback = extern "C" fn(ScriptValue, usize, usize, usize);
//                                        ↑ arg   ↑ ctx  ↑ progress ↑ completion
```

**为什么添加 `context` 参数？**

原设计没有 context，C# 静态 Thunk 无法知道调用哪个 delegate。

新设计：
- Rust 存储注册时的 context
- 触发回调时传回 context
- C# Thunk 根据 context 找到正确的 delegate

### 12.4 ScriptValue 结构

```rust
// value_bridge.rs
#[repr(C)]
pub struct ScriptValue {
    pub type_tag: u8,      // ScriptTypeTag enum
    pub error_flag: u8,    // 0 = Ok, 1 = Err
    pub reserved: u16,     // 对齐
    pub int_value: i32,
    pub float_value: f32,
```

### 12.7 C# PluginBase 实现要点

```csharp
public abstract class ScriptPlugin
{
    private unsafe void* _managerPtr;

    protected unsafe void RegisterSyncCallback(
        string name,
        Func<ScriptValue, ScriptValue> handler,
        string description = "",
        string signature = "")
    {
        // 1. GCHandle 包装
        var context = GCHandle.Alloc(handler, GCHandleType.Normal);
        var contextPtr = (nuint)GCHandle.ToIntPtr(context).ToPointer();

        // 2. UTF-8 字符串
        fixed (byte* namePtr = Encoding.UTF8.GetBytes(name + "\0"))
        fixed (byte* descPtr = Encoding.UTF8.GetBytes(description + "\0"))
        fixed (byte* sigPtr = Encoding.UTF8.GetBytes(signature + "\0"))
        {
            // 3. 注册到 Rust
            NativeMethods.scripting_register_sync_callback(
                _managerPtr, namePtr, &SyncThunk, descPtr, sigPtr, contextPtr);
        }
    }

    // 静态 Thunk（非泛型）
    [UnmanagedCallersOnly(CallConvs = new[] { typeof(CallConvCdecl) })]
    private static ScriptValue SyncThunk(ScriptValue arg, nuint context)
    {
        try
        {
            var handle = GCHandle.FromIntPtr((IntPtr)context);
            var handler = (Func<ScriptValue, ScriptValue>)handle.Target!;
            return handler(arg);  // ← 执行闭包，捕获变量可用
        }
        catch (Exception ex)
        {
            return ScriptValue.Err(ex.Message);
        }
    }
}
```

**关键点**：
1. `fixed` 语句固定 UTF-8 字符串内存
2. `GCHandleType.Normal` 允许跨 FFI 传递
3. `[UnmanagedCallersOnly]` 标记为 FFI 可调用
4. `CallConvCdecl` 匹配 Rust `extern "C"`
5. Thunk 内异常捕获，转换为 `ScriptValue.Err`

### 12.11 经验总结

1. **闭包捕获**：GCHandle + context 参数是关键
2. **FFI 签名**：统一使用 `extern "C"` + `CallConvCdecl`

```rust
        // 触发所有注册的触摸回调
        self.callbacks.lock().trigger_sync("OnTouch", script_value);
    }
    
    pub fn dispatch_key_event(&self, event: &KeyEvent) {
        let script_value = ScriptValue::from_key_event(event);
        self.callbacks.lock().trigger_sync("OnKey", script_value);
    }
    
    pub fn dispatch_size_event(&self, event: &SizeEvent) {
        let script_value = ScriptValue::from_size_event(event);
        self.callbacks.lock().trigger_sync("OnResize", script_value);
    }
}

// FFI 导出
#[unsafe(no_mangle)]
pub extern "C" fn rust_on_touch_event(engine: *mut c_void, event: *const TouchEvent) {
    let bus = unsafe { &mut *(engine as *mut EventBus) };
    bus.dispatch_touch_event(unsafe { &*event });
}

#[unsafe(no_mangle)]
pub extern "C" fn rust_on_key_event(engine: *mut c_void, event: *const KeyEvent) {
    let bus = unsafe { &mut *(engine as *mut EventBus) };
    bus.dispatch_key_event(unsafe { &*event });
}

#[unsafe(no_mangle)]
pub extern "C" fn rust_on_size_event(engine: *mut c_void, event: *const SizeEvent) {
    let bus = unsafe { &mut *(engine as *mut EventBus) };
    bus.dispatch_size_event(unsafe { &*event });
}
```

### 13.11 设计要点总结

1. **NAPI 作为桥梁**：ArkTS → C Bridge → Rust FFI
2. **事件结构统一**：使用 `#[repr(C)]` 结构，C/Rust 共享
3. **回调复用**：EventBus 使用 Phase 1 的 CallbackRegistry
4. **Surface opaque 类型**：OH_NativeWindow 作为 opaque pointer，Rust 不直接访问内部
5. **生命周期同步**：HarmonyOS Ability 状态映射到 Rust lifecycle events

#### 关键实现要点

**Rust 端**：
- `NativeWindowContext` 管理 Surface 回调注册
- `EventBus` 分发触摸/键盘/尺寸事件到 C# 脚本
- `HarmonyEngine` 整合所有组件

**C Bridge**：
- NAPI 导出 `initEngine/shutdownEngine/onTouchEvent/onKeyEvent/onResize`
- 从 ArkTS 参数构造 Rust 事件结构
- 调用 Rust FFI 函数传递事件

**数据流**：
```
ArkTS XComponent.onTouch()
    ↓ NAPI (JS → C)
hezhou_native.cpp:OnTouchEvent()
    ↓ 构造 TouchEvent 结构
harmony_on_touch_event() [Rust FFI]
    ↓ EventBus.dispatch_touch_event()
CallbackRegistry.trigger_sync("OnTouch", ScriptValue)
    ↓ [C# Thunk]
HarmonyOSPlugin.OnTouch 回调执行
```
